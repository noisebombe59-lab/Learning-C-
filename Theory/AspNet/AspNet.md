﻿1. Первоисточник (Твой Код)
Все начинается с того, что ты пишешь классы, контроллеры и логику на C#. Твой код — это «двигатель» и «правила игры».

2. OpenAPI — «Технический Чертеж» (Автоматический паспорт)

Что это: Документ в формате JSON, который система генерирует на основе твоего кода.

Роль: Это «зеркало» твоего приложения. Если ты добавил в код поле Email, OpenAPI тут же запишет это в чертеж.

Зачем: Чтобы внешние программы (и пользователи) знали, по какому адресу стучать и в каком формате передавать данные.

3. Scalar — «Лицо» (Визуализация чертежа)

Что это: Красивый веб-интерфейс, который «читает» OpenAPI.

Роль: Превращает скучный JSON-текст в интерактивную страницу с кнопками «Try it out». Пользователь смотрит в Scalar, чтобы понять, как отправить тебе запрос.

4. Эндпоинт — «Точка входа»

Это конкретный метод в твоем контроллере, привязанный к адресу (URL) и глаголу (GET, POST, PUT, DELETE). Это «ручка», за которую дергает пользователь.

5. Цепочка обработки запроса (Путь «Туда»):

Пользователь: Нажимает кнопку в Scalar.

API (Диспетчер): Принимает запрос, сверяет его с чертежом и делает Binding (распаковку данных из текста JSON в объекты C#).

Контроллер (Начальник отдела): Получает «распакованный» объект и передает эстафету дальше.

Сервис (Мастер-логик): Здесь живет вся «черная работа». Тут ты используешь LINQ, проводишь расчеты и манипуляции с данными.

6. Цепочка возврата (Путь «Обратно»):

Сервис: Отдает готовый результат (объект C#) Контроллеру.

Контроллер: Говорит системе: «Готово, отправляй».

API (Упаковщик): Берет твой объект C#, автоматически упаковывает его в JSON-текст и отправляет пользователю в браузер.

Итог одной фразой:
Ты пишешь код → API создает чертеж (OpenAPI) → Пользователь по чертежу шлет запрос → API распаковывает данные → Сервис делает магию → API упаковывает результат в JSON и отдает его обратно.

7. Модели (Чертежи объектов)
Модель — это класс в C#, который описывает структуру данных для базы. Это «шаблон», по которому создаются реальные записи.

Типизация: Для каждого свойства выбирается строгий тип (например, decimal для денег, чтобы сохранить точность, или long для очень больших чисел).

Защита от Null (string.Empty): По умолчанию строковые свойства могут быть равны null (абсолютная пустота в памяти). Чтобы программа не «падала» при попытке прочитать пустую строку, полям сразу присваивается значение string.Empty (пустой лист).

8. Ключи (Идентификаторы)
Ключи позволяют базе данных отличать записи друг от друга и связывать их между собой.

Primary Key (ID): Первичный ключ. Это уникальный «паспортный номер» каждой строки в таблице. Обычно это целое число, которое база данных увеличивает автоматически (1, 2, 3...).

Foreign Key (Внешний ключ): Это поле в «дочерней» таблице, которое хранит ID «родителя». Например, в таблице Comments поле StockId указывает, к какой конкретно акции привязан этот отзыв. Без этого номера база не поймет, чей это комментарий.

9. Навигационные свойства (Navigation Properties)
Это специальные ссылки внутри C#-классов, которые позволяют программе легко перемещаться между связанными объектами без написания сложных поисковых запросов. Это своего рода «рельсы» в коде.

Коллекция (у Родителя): Позволяет главному объекту видеть список всех своих «детей».

Пример: public List<Comment> Comments { get; set; } внутри класса Stock.

Ссылка (у Ребенка): Позволяет дочернему объекту напрямую обращаться к данным своего родителя.

Пример: public Stock? Stock { get; set; } внутри класса Comment.

10. Тип связи: Один-ко-многим (One-to-Many)
Самый распространенный тип отношений. Один родительский объект (Акция) может иметь неограниченное количество дочерних объектов (Комментариев), но каждый дочерний объект принадлежит строго одному родителю.

Итог:
Правильное именование (например, Id для ключа и StockId для связи) позволяет библиотекам вроде Entity Framework автоматически понимать структуру твоей базы данных, превращая обычный код в мощную систему хранения.

11. Зависимости (Dependencies) — «Ящик с инструментами»
Это всё, что ваш проект использует «извне», чтобы не писать велосипеды.

NuGet-пакеты: Библиотеки, скачанные из глобального хранилища. Например, Entity Framework Core (для работы с БД) или Scalar/Swagger (для документации).

Frameworks: Системные библиотеки самого .NET.

Зачем: Если в коде что-то подчеркнуто красным (например, DbContext или ControllerBase), значит, в «Зависимостях» не хватает нужного инструмента (пакета).

12. Миграции (Migrations) — «История чертежей БД»
Это мост между вашим C#-кодом и таблицами в SQL Server.

Суть: Когда вы меняете класс (например, добавили поле в Stock.cs), база данных об этом не знает. Миграция — это файл-инструкция, который говорит базе: «Добавь колонку X в таблицу Y».

Процесс:

Изменили модель (класс).

Выполнили команду Add-Migration — создался файл с датой в названии (тот самый «чертеж»).

Выполнили Update-Database — изменения применились к реальной базе.
