1. Архитектура запроса (Request Flow)
Твой код (C#): Двигатель и логика.

OpenAPI: Авто-чертеж (JSON) твоего API для внешнего мира.

Scalar: Визуальный интерфейс («лицо») для тестов и кнопок «Try it out».

Эндпоинт: Конкретный метод контроллера (URL + Глагол GET/POST).

Путь: Пользователь (Scalar) → API (Распаковка JSON в C#) → Контроллер → Сервис (Логика/LINQ) → Контроллер → API (Упаковка C# в JSON) → Пользователь.

2. Базы данных и Модели (EF Core)
Модель: C#-класс — шаблон для таблицы. Используй decimal для денег и string.Empty для защиты от null.

Primary Key (ID): Уникальный паспорт записи.

Foreign Key (StockId): Ссылка в «ребенке» на ID «родителя».

Навигационные свойства: «Рельсы» в коде. List<Child> у родителя и Parent? у ребенка.

Связь 1:N: Одна акция — много комментариев.

3. Структура проекта
Зависимости (NuGet): Внешние инструменты (библиотеки).

Миграции: История изменений БД. Add-Migration (чертеж) → Update-Database (стройка).

appsettings.json: Текстовый конфиг (адреса баз, пароли) без пересборки кода.

4. Внедрение зависимостей (DI)
Суть: Не создавай инструменты сам (new), а проси их в конструкторе.

Регистрация (Program.cs): builder.Services.Add... (запись в инвентарь).

Инъекция: Получение через параметры конструктора.

5. Сроки жизни (Lifetimes)
Transient (Стаканчик): Новый объект на каждый «чих».

Scoped (Тарелка): Один объект на один HTTP-запрос (от клика до ответа). Стандарт для DbContext.

Singleton (Кулер): Один объект на всё время работы сервера для всех юзеров.
