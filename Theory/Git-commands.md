1. Начало работы (Инициализация)
Когда проект только создается «с нуля» на компьютере:

        git init — превращает папку в репозиторий.
        
        ren .gitignore.txt .gitignore — правильное переименование файла игнорирования (в Windows через консоль).
        
        git remote add origin <link> — привязка к пустому репозиторию на GitHub.
        
        git branch -M main — установка имени основной ветки.

2. Ежедневный цикл (Workflow)
Стандартная цепочка действий при написании кода:

        git status — проверить, что именно изменилось.
        
        git add . — подготовить все изменения к сохранению.
        
        git commit -m "описание" — создать точку восстановления (коммит).
        
        git push — отправить изменения в облако.

3. Работа над ошибками (Fixing)
Если что-то пошло не так с файлами или игнорированием:

        Сброс отслеживания: git rm -r --cached . — заставляет Git «забыть» всё и перечитать .gitignore.
        Помогает, если папки bin или obj уже попали в историю.
        
        Отмена изменений (до коммита): git restore <file> — вернуть файл к состоянию последнего коммита (удалить локальные правки).
        
        Исправление последнего сообщения: git commit --amend -m "новое сообщение" — если опечатался в тексте последнего коммита.

5. Синхронизация и решение конфликтов
Самое важное, когда работаешь с GitHub (твой недавний случай):

        git pull origin main — забрать изменения из облака и объединить со своими.
        
        git pull --rebase — более «чистый» способ: временно убирает твои коммиты, подкладывает под них серверные и возвращает твои сверху.
        
        Если возник конфликт (Detached HEAD):
        
        Отредактировать файлы (удалить <<<<<<< и =======).
        
        git add .
        
        git rebase --continue — завершить процесс объединения.

5. Полезные команды «на всякий случай»
        git log --oneline — компактный список всех прошлых коммитов (твоя история).

git remote -v — проверить, к какому адресу привязан проект.

git push -f origin main — «Силовой метод». Заменяет всё на сервере твоим кодом. Использовать крайне осторожно!
