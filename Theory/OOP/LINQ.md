LINQ — Философия потока
Суть: Превращает рутину циклов foreach в описание «конвейера» данных.
Самая частая цепочка в работе:
Источник ➔ Where (фильтр) ➔ Join (связь) ➔ GroupBy (папки) ➔ Select (итоги) ➔ OrderBy (красота) ➔ ToList (финал).

I. Архитектурные основы
1. Декларативный стиль (Declarative Programming)
Суть: Подход, при котором код описывает «что» нужно получить, а не «как» (пошаговая реализация циклов и проверок скрыта внутри методов LINQ).

        C#
        // Императивный стиль (КАК делать):
        var result = new List<Goods>();
        foreach (var item in _allGoods) {
            if (item.Price > 100) result.Add(item);
        }

        // Декларативный стиль (ЧТО получить):
        var result = _allGoods.Where(g => g.Price > 100);
Когда нужен: Чтобы код читался как текст, а не как набор инструкций для процессора.

2. Отложенное выполнение (Deferred Execution)
Суть: Механизм, при котором запрос не выполняется в момент создания. Он «срабатывает» только тогда, когда вы начинаете перечислять элементы (через foreach, .ToList(), .Count()).

        C#
        var query = _allGoods.Where(g => g.Price > 5000); // Поиск еще не начался
        // ... здесь можно добавить еще условий ...
        var list = query.ToList(); // Только здесь данные потекли через фильтр в память
Когда нужен: Позволяет строить запрос по кусочкам (добавлять фильтры) до того, как данные будут реально скачаны.

3. IEnumerable
Суть: Интерфейс-контракт, который позволяет перечислять элементы коллекции. Все методы LINQ — это методы расширения для этого интерфейса. Работает через итератор: данные подтягиваются по одному, «лениво».

        C#
        // 1. Любой список (List) реализует IEnumerable
        IEnumerable<string> names = new List<string> { "TV", "Smartphone", "Tablet" };
        
        // 2. Благодаря интерфейсу мы можем вызывать методы LINQ
        IEnumerable<string> filteredNames = names.Where(n => n.Length > 2);
        
        // 3. Данные подтягиваются только при переборе
        foreach (string name in filteredNames) {
            Console.WriteLine(name); 
        }
Когда нужен: Базовый «клей» для всех методов LINQ. Позволяет работать с данными, не загружая весь список в память сразу.

🛠️ II. Базовые операции (Фильтрация и Поиск)
1. Where (Фильтрация)
Суть: Отбирает элементы, соответствующие заданному условию (предикату). На выходе — тот же тип объектов, но меньше по количеству.

        C#
        var electronics = _allGoods.Where(g => g.Category == "Electronics" && g.Price < 1000);
Когда нужен: Когда из общего мешка нужно достать только то, что подходит под правило.

2. Select (Проекция / Трансформация)
Суть: Трансформирует элементы одного типа в другой или выбирает конкретные поля («переодевание» данных).

        C#
        // Трансформация объекта в строку
        var names = _allGoods.Select(g => g.Name);
        
        // Создание анонимного объекта (только имя и цена)
        var simpleItems = _allGoods.Select(g => new { g.Name, g.Price });
Когда нужен: Чтобы не таскать за собой лишние поля или подготовить данные для вывода.

3. First / FirstOrDefault (Поиск одного)
Суть: Находит первый элемент по условию. FirstOrDefault вернет null, если ничего не найдено.

        C#
        var item = _allGoods.FirstOrDefault(g => g.Name == "iPhone");
Когда нужен: Когда нужен конкретный уникальный объект, а не список.

4. Кванторы (Any, All, Contains)
Суть: Методы, возвращающие bool (проверка условий на всей коллекции). Any — есть ли хоть один? All — все ли такие?

        C#
        bool hasDebt = _allGoods.Any(g => g.Stock == 0); // Есть ли хоть один пустой?
        bool allValid = _allGoods.All(g => g.Price > 0); // Все ли цены положительные?
Когда нужен: Для быстрых логических проверок коллекции перед началом расчетов.

📊 III. Сортировка и Агрегация
1. OrderBy / ThenBy (Сортировка)
Суть: Упорядочивание данных. ThenBy используется для вторичной сортировки («внутри» первой).

        C#
        var sorted = _allGoods
            .OrderBy(g => g.Category)          // Сначала по категории
            .ThenByDescending(g => g.Price);   // Внутри категории по цене (от дорогих)
Когда нужен: Для приведения хаоса в читаемый вид (по алфавиту, по цене и т.д.).

2. Агрегатные функции (Sum, Max, Count)
Суть: Схлопывает всю коллекцию в одно итоговое значение (число).

        C#
        var totalValue = _allGoods.Sum(g => g.Price * g.Stock);
        var maxPrice = _allGoods.Max(g => g.Price);
Когда нужен: Когда важны не сами объекты, а их итоговые показатели (общая сумма, средний чек).

🚀 IV. Продвинутые манипуляции
1. GroupBy (Группировка / Папки)
Суть: Разрезает список на «папки» по ключу. У каждой папки есть Key (название) и содержимое (список внутри).

        C#
        var report = _allGoods
            .GroupBy(g => g.Category)
            .Select(group => new {
                CategoryName = group.Key,
                Count = group.Count(),
                AvgPrice = group.Average(x => x.Price)
            });
Когда нужен: Когда нужно «схлопнуть» дубликаты в структуру (например, продажи по месяцам).

2. Join (Соединение / Сваха)
Суть: Объединение двух разных коллекций по совпадающему ключу. Если у объекта нет пары во втором списке — он удаляется из результата.

        C#
        var discounts = new Dictionary<string, decimal> { { "Electronics", 0.15m } };
        
        var discountedGoods = _allGoods.Join(
            discounts,
            g => g.Category,          // Ключ первой коллекции
            d => d.Key,               // Ключ второй (словаря)
            (g, d) => new { g.Name, SalePrice = g.Price * (1 - d.Value) }
        );
Когда нужен: Когда нужно быстро «склеить» две таблицы в одну плоскую строку.

3. SelectMany (Распаковка / Flattening)
Суть: Если внутри элементов одного списка лежат другие списки, SelectMany «вскрывает» их и высыпает всё содержимое в один общий плоский поток. Границы «папок» стираются.

        C#
        var departments = new List<Department> {
            new { Name = "IT", Staff = new[] { "Ivan", "Oleg" } },
            new { Name = "HR", Staff = new[] { "Anna" } }
        };
        
        // Результат: ["Ivan", "Oleg", "Anna"]
        var allStaff = departments.SelectMany(d => d.Staff);
Когда нужен: Только если есть вложенность (список в списке). Без неё — бесполезен.

4. GroupJoin (Иерархия / Left Join)
Суть: Берет элемент из левого списка и «пришивает» к нему целую папку (группу) совпадений из правого. Если совпадений нет — элемент не исчезает, просто папка будет пустой.

        C#
        var cats = new[] { "Electronics", "Books" };
        var prods = new[] { new { Name = "Phone", Cat = "Electronics" } };
        
        var report = cats.GroupJoin(
            prods,
            cat => cat,          // Ключ категории
            prod => prod.Cat,    // Ключ товара
            (cat, prodGroup) => new { 
                Category = cat, 
                Items = prodGroup // prodGroup — это "папка" со всеми товарами категории
            }
        );
        // Результат: Electronics (1 товар), Books (0 товаров — Books НЕ удалились!)
Когда нужен: Для отчетов, иерархий (Один-ко-Многим) и когда важно сохранить «пустые» категории, которые обычный Join бы выкинул.

5. Множества (Distinct, Except, Union, Intersect)
Суть: Операции сравнения коллекций как математических множеств.

        C#
        var listA = new[] { "TV", "Phone" };
        var listB = new[] { "Phone", "Laptop" };
        
        var uniqueInA = listA.Except(listB);    // "TV" (в А, нет в Б)
        var shared = listA.Intersect(listB);    // "Phone" (в обоих)
        var combined = listA.Union(listB);      // "TV", "Phone", "Laptop" (без дублей)
        var distinct = listA.Distinct();        // Убрать дубликаты
🛡️ V. Техника и Безопасность
1. Skip / Take (Пагинация)
Суть: Отрезает кусок списка. Skip — пропустить, Take — взять.

        C#
        int pageSize = 10;
        int pageNumber = 3;
        var page = _allGoods.Skip((pageNumber - 1) * pageSize).Take(pageSize);
Когда нужен: Чтобы не грузить пользователю миллион строк сразу, а выдавать порциями.

2. Zip (Замок-молния)
Суть: Склеивает два разных списка строго по индексу (первый с первым).

        C#
        var names = new[] { "Item1", "Item2" };
        var codes = new[] { "A-01", "B-02" };
        var zipped = names.Zip(codes, (n, c) => $"{n}: {c}"); 
Когда нужен: Когда данные лежат в двух параллельных массивах.

3. Лишние преобразования
Определение: Избыточный вызов .ToList() или .ToArray() внутри цепочки, что приводит к ненужному копированию данных в память.
Когда нужен: Только в самом конце цепочки, когда работа со «стримом» (IEnumerable) закончена и нужны реальные данные в памяти.
