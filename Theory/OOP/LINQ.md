LINQ (Language Integrated Query) — это единый стандарт написания запросов в C#.

«Это технология в .NET, которая позволяет писать строго типизированные запросы к разным источникам данных (объектам, БД, XML) прямо на языке C#. 
Она делает код короче и читаемее за счет декларативного подхода и предоставляет мощные инструменты: фильтрацию, сортировку и группировку «из коробки».

1. Декларативный стиль (ЧТО, а не КАК)
Императивный код (старый стиль): Ты диктуешь компьютеру каждый шаг. «Возьми список, создай новый, запусти цикл, проверь условие, если ок — добавь в новый список».

Декларативный код (LINQ): Ты просто описываешь результат. «Дай мне всех пользователей, которым больше 18 лет, отсортированных по имени». Ты не заботишься о том, как именно внутри работает цикл.

2. Отложенное выполнение (Deferred Execution)
Это самая важная техническая деталь. Когда ты пишешь var result = list.Where(x => x > 5);, запрос еще не выполнился.
LINQ просто «запомнил» твое желание. Реальная работа начнется только тогда, когда ты начнешь перебирать result (например, в foreach) или вызовешь методы типа .ToList(), .ToArray(), .Count().

Зачем это нужно? Это экономит ресурсы. Ты можешь настроить цепочку из 10 фильтров, но данные будут обработаны ровно один раз в самом конце.

3. Универсальный интерфейс IEnumerable<T>
LINQ работает не с конкретными классами, а с интерфейсом «перечислитель». Если объект можно перебрать (как список или массив), значит, к нему можно применить LINQ. Это делает твой код гибким: сегодня у тебя List, завтра ты заменил его на HashSet, а LINQ-запросы продолжают работать без изменений.

У LINQ есть два синтаксиса. Ты должен знать оба, так как в проектах они встречаются вперемешку.

Method Syntax (Методы расширения) — Самый популярный. Похож на цепочку вызовов.

    C#
    var seniors = employees.Where(e => e.Age > 50).OrderBy(e => e.Name);
    
  Query Syntax (Похож на SQL) — Удобен для очень сложных запросов (например, объединение нескольких таблиц).
    
    C#
    var seniors = from e in employees
                  where e.Age > 50
                  orderby e.Name
                  select e;
