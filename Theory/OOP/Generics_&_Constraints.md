Generics & Constraints (Обобщения)
Суть: «Когда я пишу код, который работает с любыми типами, но я хочу ограничить их (например, "только для классов, у которых есть ID")».

1. Синтаксис Обобщений (Generics)
Суть: Мы не фиксируем конкретный тип (например, int), а используем параметр типа (обычно букву T). Это позволяет избежать дублирования кода для разных типов данных.

        C#
        // Обобщенный класс: Универсальный контейнер
        public class Box<T> 
        {
            private T _content;
            public void Store(T item) => _content = item;
            public T Retrieve() => _content;
        }
        
        // Обобщенный метод: Работает с типом "на лету"
        public void PrintInfo<T>(T item) 
        {
            Console.WriteLine($"Тип: {typeof(T)}, Значение: {item}");
        }
Когда нужен: Когда логика работы с данными (хранение, сортировка, передача) одинакова, а сами типы данных — разные.

2. Базовые ограничения (where T : ...)
Суть: По умолчанию T — это черный ящик. Ограничения (Constraints) открывают этот ящик, разрешая вызывать методы или создавать объекты.

        C#
        // where T : class — Только ссылочные типы (можно T = null)
        // where T : struct — Только значимые типы (числа, структуры)
        // where T : new() — У типа обязан быть пустой конструктор (можно писать new T())
        
        // where T : IEntity — Самое полезное: гарантирует наличие методов интерфейса
        public void Process<T>(T item) where T : IEntity 
        {
            Console.WriteLine(item.Id); // Работает, так как T обязан быть IEntity
        }
Когда нужен: Когда тебе нужно не просто «что-то» хранить, а совершать с типом T конкретные действия (вызывать методы, создавать экземпляры).

3. Множественные ограничения
Суть: Комбинирование правил для T. Важен строгий порядок: сначала тип (class/struct), затем интерфейсы, в конце — new().

        C#
        // Правильный порядок: class -> интерфейсы -> new()
        public class Processor<T> where T : class, ICloneable, new() 
        {
            public T CloneNew() => new T(); // Работает благодаря new()
        }
Когда нужен: Для создания сложных универсальных компонентов (например, кэша, который работает только с объектами, которые можно клонировать).

4. Работа с default
Суть: В обобщенном коде ты не знаешь, что такое «пусто» (0 или null). Ключевое слово default само подбирает безопасное «нулевое» значение для любого типа.

        C#
        public T GetDefaultValue<T>()
        {
            return default; // Для int вернет 0, для string — null
        }
        
        // Вызов:
        int x = GetDefaultValue<int>(); 
        string s = GetDefaultValue<string>(); 
Когда нужен: Когда нужно инициализировать переменную типа T или вернуть «ничего», не зная заранее, ссылка это или число.

🛠 Итоговая шпаргалка архитектора:
Нужна универсальность без потери типов? — Используй Generics.

Нужно создавать объекты new T()? — Добавь ограничение where T : new().

Нужно вызвать метод у T? — Добавь ограничение where T : IInterface.

Нужно обнулить T? — Используй default.
