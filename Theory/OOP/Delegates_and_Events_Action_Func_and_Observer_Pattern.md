Delegates, Events & Lambdas
Суть: «Когда я хочу, чтобы один объект мог сообщить другим о том, что что-то случилось (например, "Транзакция завершена"), не зная ничего о тех, кто его слушает».

1. Делегаты (Указатели на методы)
Суть: Это тип данных, который хранит ссылку на метод. Делегат определяет «форму» метода (что принимает и что возвращает), а затем в него можно положить любой подходящий метод.

        C#
        // 1. Объявление типа делегата (сигнатура)
        public delegate void LogHandler(string message);
        
        public class Logger {
            public void ToConsole(string msg) => Console.WriteLine($"[LOG]: {msg}");
        }

        // 2. Использование:
        var logger = new Logger();
        LogHandler handler = logger.ToConsole; // Привязываем метод
        handler("Система запущена");           // Вызов через делегат
Когда нужен: Когда методу нужно передать не просто данные (число, строку), а целое действие (алгоритм).

2. Встроенные делегаты (Action, Func)
Суть: Чтобы не плодить свои типы делегатов, в C# используют готовые шаблоны.

Action — если метод ничего не возвращает (void).

Func — если метод возвращает значение (последний тип в списке — всегда результат).

    C#
    Action<string> print = (msg) => Console.WriteLine(msg); // Void-метод
    Func<int, int, int> multiply = (a, b) => a * b;        // Принимает 2 int, возвращает int
    Predicate<int> isPositive = (n) => n > 0;               // Возвращает bool
Когда нужен: В 99% рабочих задач. Позволяет писать лаконичный код без лишних объявлений delegate.

3. События (Events)
Суть: Безопасная «обертка» над делегатом. Реализует инкапсуляцию: внешний код может только подписываться (+=) или отписываться (-=), но не может вызвать событие сам или стереть чужие подписки.

        C#
        public class TemperatureSensor {
            // Событие на базе Action. Извне нельзя сделать = null.
            public event Action<double>? TemperatureChanged;
        
            public void Measure(double temp) {
                // ?.Invoke проверяет, есть ли хотя бы один подписчик
                TemperatureChanged?.Invoke(temp); 
            }
        }
Когда нужен: Для реализации паттерна «Издатель — Подписчик». Гарантирует, что только владелец класса может «нажать на кнопку» оповещения.

4. Паттерн Наблюдатель (Observer)
Суть: Создание системы подписок. Один источник (Сенсор) уведомляет множество независимых слушателей (Экран, Сигнализация).

        C#
        var sensor = new TemperatureSensor();
        var display = new Display();
        var alarm = new Alarm();
        
        // СВЯЗЫВАЕМ объекты (Подписка)
        sensor.TemperatureChanged += display.ShowTemperature;
        sensor.TemperatureChanged += alarm.CheckAlert;
        
        sensor.Measure(45.0); // Уведомление уйдет обоим подписчикам сразу
Когда нужен: Когда изменение в одном объекте должно вызывать автоматическую реакцию в других, при этом объекты не должны «знать» друг о друге напрямую.

5. Лямбда-выражения (Анонимные методы)
Суть: Метод без имени, который создается «на лету» прямо там, где он нужен.

Одиночная: x => x * x (авто-return).

Блочная: x => { if(x > 0) return true; return false; } (нужен {} и return).

Без параметров: () => Console.WriteLine("Hi").

Несколько параметров: (a, b) => a + b.

    C#
    // Пример: фильтрация списка через лямбду
    var heavyCargos = cargos.Where(c => c.Weight > 500); 
Когда нужен: В LINQ, при подписке на события или везде, где метод слишком мал и специфичен, чтобы давать ему отдельное имя в классе.

🛠 Итоговая шпаргалка архитектора:
Хочешь передать действие как параметр? — Используй Action или Func.

Нужно создать систему уведомлений? — Используй event.

Нужно быстро описать логику «на месте»? — Используй Лямбду.

Хочешь защитить список подписчиков? — Ключевое слово event обязательно (вместо обычного делегата).
